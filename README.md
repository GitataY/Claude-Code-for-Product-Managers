# Claude Code for Product Managers
 
**Original product thinking, teardowns, and PM artifacts built with AI-assisted workflows.**
 
This repo documents my journey learning AI-powered product management through the [Claude Code for Product Managers](https://ccforpms.com/) course by [Carl Vellotti](https://www.linkedin.com/in/carlvellotti/), alongside my own independent product work.
 
Everything here is my own analysis and writing. Course materials are not included - only the artifacts I produced.
 
---
 
## Repository Structure
 
```
Product-Outputs/
├── fundamentals/        # Core PM skills built with Claude Code
├── teardowns/           # Deep-dives into real products
├── prds/                # Product Requirements Documents
├── case-studies/        # Business-framed product analyses
├── user-stories/        # User stories with acceptance criteria
├── test-plans/          # QA thinking and test scenarios
└── blog-drafts/         # Long-form writing on PM and AI
```
 
## How I Work
 
I use [Claude Code](https://claude.ai) as part of my PM workflow - not to replace the thinking, but to accelerate execution. My practice includes:
 
- Writing PRDs, user stories, and tech specs with AI-assisted drafting
- Product teardowns of real products
- Spec-driven development principles
- Test planning as code - structuring QA artifacts as markdown
